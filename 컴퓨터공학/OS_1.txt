운영체제란?
하드웨어를 전반적으로 관리하는 관리자 역할
즉, 운영 체제는 응용 프로그램이 요청하는 시스템 리소스를 효율적으로 분배하고, 지원하는 소프트웨어를 말한다.

1. 시스템 자원 관리자
2. 사용자와 컴퓨터간의 커뮤니케이션 지원( 사용자가 컴퓨터를 어떻게 써야하는지)
3. 컴퓨터 하드웨어와 응용 프로그램을 제어

운영체제와 응용 프로그램간의 관계
1. 응용 프로그램을 관리
2. 시스템 자원을 관리
3. 사용자와 컴퓨터간의 커뮤니케이션 지원
목표: 사용자가 사용하는 응용 프로그램이 효율적으로, 적절하게 동작하도록 지원한다.
(1950)배치(Batch) 처리 시스템?
자동으로 다음 응용 프로그램이 이어서 실행될 수 있도록 하는 시스템
단점: 컴퓨터 응답 시간이 오래 걸릴 수 있다.(스케줄러 부재)

(1960)시분할 시스템, 멀티태스킹
프로그램을 시간대로 분할하여 여러개의 프로그램을 작동시킬수 있도록 하는 시스템 (CPU에서 처리하는 시간을 잘게 자름)
*다중 사용자 지원

멀티 태스킹: 단일 CPU에서, 여러 응용 프로그램의 병렬 실행을 가능케 하는 시스템.

보통 시분할 시스템 = 멀티 태스킹

결과적으로 사용자가 느낄 경우, 여러 응용 프로그램이 동시에 실행되는 것처럼 보인다.

1970년대
제대로된 운영체제의 출현
현대 운영체제 기술 확립
UNIX OS 개발
특징 :
현대 운영체제의 기본 기술을 모두 포함한 최초의 운영체제
  멀티 태스킹, 시분할 시스템, 멀티 프로그래밍

1980년대
개인용 컴퓨터 시대
1980년대 이전: 시분할 시스템으로 인해 대형 컴퓨터를 여러 명이 접속해서 사용(UNIX)
1980년대 ~: Personal Computer (PC)
CLI --> GUI 로 변경되는 시점

1990년대
응용 프로그램 시대 by GUI 환경, 개인용 컴퓨터의 사용자 증대
액셀, 워드 프로세서 등등
Windows OS 대중화
Linux 와 같은 오픈소스 운동 활성화 시작
2. 내트워크 기술 발전 - 월드와이드웹(WWW) 인터넷 대중화

2000년대
1. 오픈소스 활성화
Linux, Apache(웹서버), MySQL(데이터베이스) 등등...
2. 가상 머신, 대용량 병렬 처리 등 활성화
